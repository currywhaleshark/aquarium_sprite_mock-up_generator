경골어류 아가미덮개 형태와 렌더링 보고서

현재 구현의 문제는 “아가미덮개가 너무 튄다”가 맞다. 실제 경골어류에서는 덮개가 머리 표면과 같은 조직 위에 얹힌 얕은 판처럼 이어지고, 몸통과는 후두부/목 경계에서 분리되어 읽힌다. 따라서 별도 색의 배지처럼 보이면 해부학적으로도, 렌더링 관습으로도 어색하다.

작성일: 2026-06-08 범위: 경골어류/진골어류 중심 제외: 상어류의 다중 외부 아가미틈 대상 파일: fish_body_toon.gdshader

요약 판단

형태

아가미덮개는 독립 장식이 아니라 머리 뒤쪽 측면의 얕은 골판 구조다. 앞쪽은 프리오퍼클 경계로 머리와 이어지고, 뒤쪽/아래쪽 자유연만 어두운 배출구로 읽힌다.

렌더링

실시간/스타일라이즈드 렌더링에서는 실루엣을 크게 바꾸지 않는 한 표면에 칠한다. 알베도 어둡힘, 얇은 선, 노멀/범프 힌트를 쓰고, 별도 돌출 메시지는 근접 컷신용에 가깝다.

현재 수정 방향

판 전체를 어둡게 강조하기보다 머리색과 섞고, 후방 슬릿만 가장 또렷하게 둔다. 몸과 머리의 분리는 아가미덮개 내부가 아니라 목/몸통 연결부에서 별도 처리한다.

권장 기본값: operculum_ridge 기본 시각 강도는 현재보다 낮추고, 판 다크닝은 “색 변경”이 아니라 “현지 표면색의 5-12% 음영” 정도로 제한한다. 시선은 전방 J자 선보다 후방 출구에 조금 더 모이게 한다.

실제 구조: 머리와 이어진 판, 뒤쪽만 열리는 출구

머리 표면 (덮개와 같은 톤) 몸통 (목에서 분리) 오퍼클 판 음영 프리오퍼클 J 경계 후방 아가미 출구

해부학적 읽기

경골어류의 겉보기 아가미 구조는 여러 개의 외부 틈이 아니라, 아가미방을 덮는 골성 덮개다. 보이는 구성은 대체로 프리오퍼클, 오퍼클, 서브오퍼클, 인터오퍼클의 opercular series로 이해하면 된다.

프리오퍼클은 앞쪽 경계와 감각관을 품는 chevron/J자형 뼈로 읽히고, 오퍼클은 덮개 대부분을 지지하는 얇고 넓은 판이다. 뒤쪽 가장자리는 열릴 수 있는 자유연이고, 실제 물 배출은 후방/복측 opercular opening을 통해 보인다.

그래서 옆모습에서 가장 자연스러운 표현은 “머리색과 거의 같은 넓은 판 + 약한 전방 J자 경계 + 더 어두운 후방 출구 + 선택적인 하단 보조선”이다.

연결 방식: 힌지는 앞쪽, 숨구멍은 뒤쪽

구조 실제 역할 렌더링에서 보이는 신호 현재 셰이더에 주는 의미
프리오퍼클 앞쪽 경계. hyomandibular/quadrate 주변과 연결되며 감각관/공을 가질 수 있다. 눈 뒤-볼 뒤쪽을 따라 내려가는 완만한 J자 또는 초승달 선. 가장 진한 선이면 안 된다. 얇고 부드럽게, 머리 표면에 묻혀야 한다.
오퍼클 덮개 대부분을 이루는 넓고 얇은 판. 별도 색 면이 아니라 같은 머리색에서 살짝 낮은 명도/거친 하이라이트. 고정 색으로 칠하지 말고 col * shade처럼 현지 표면색을 조절한다.
서브오퍼클/인터오퍼클 하단 덮개와 막 지지. 작은 스프라이트에서는 생략 가능. 크게 보일 때만 하단의 짧은 보조선. 기본값에서는 거의 안 보이게 두고, 큰 어종/근접 뷰에서만 활성화한다.
후방 자유연 호흡 시 열리며 물이 빠져나가는 출구. 덮개 뒤쪽의 좁고 어두운 곡선. 열림 값이 크면 폭/명암이 조금 증가. 가장 또렷한 실루엣 신호. 다만 몸통 경계선과 합쳐지지 않게 위치를 살짝 앞에 둔다.
몸통-머리 경계 두개골/후두부와 몸통 쉘이 만나는 영역. 아가미덮개 내부가 아니라 목 뒤쪽의 별도 완만한 분리선/그림자. 아가미덮개 셰이더와 별도 파라미터로 관리하는 편이 안전하다.

왜 지금 튀어 보이는가

1. 판과 선의 대비가 같은 층에 있다

실제 덮개는 넓은 판보다 뒤쪽 출구가 더 잘 읽힌다. 현재처럼 판 전체 어둡힘, 전방 seam, 후방 slit이 모두 강하면 장식 패널처럼 보인다. 특히 작은 스프라이트에서는 “아가미덮개”가 아니라 “검은 무늬”가 먼저 보인다.

2. 머리 전용 게이트가 약하다

최근 스펙은 is_head && operculum_enabled를 의도했지만, 현재 셰이더의 조건은 operculum_enabled만 사용한다. 몸통 재질에도 같은 uniform이 켜지면 아가미덮개 마스크가 몸통 앞부분까지 읽혀 “머리와 몸통이 둘 다 칠해진” 인상을 만들 수 있다.

3. 몸과의 분리가 아가미덮개에 떠넘겨졌다

사용자가 원하는 핵심은 “머리+아가미덮개는 자연스럽게 연결, 몸과는 분리”다. 그러면 아가미덮개 판 내부를 강하게 그리는 대신, 목 경계/후두부 접합부에 별도 약한 occlusion 또는 좁은 neck seam을 주는 편이 맞다.

4. 위치가 너무 중앙 장식처럼 보일 수 있다

덮개는 눈 바로 뒤 장식이 아니라 볼 뒤쪽에서 목 앞까지 이어지는 구조다. 전방 경계는 눈을 침범하지 않고, 후방 출구는 몸통 시작선보다 살짝 앞에 있어야 한다.

주의: 상어처럼 여러 개의 외부 아가미틈을 그리는 접근은 이번 범위에서 제외한다. 경골어류에서는 한쪽에 여러 줄의 검은 틈을 만들수록 실제 구조에서 멀어진다.

권장 셰이더 디자인

레이어 우선순위

  1. 머리/몸통 기본색과 countershading.
  2. 어종 고유 패턴. 단, 눈 뒤 아가미덮개 영역에서는 너무 강한 무늬를 살짝 눌러도 된다.
  3. 아가미덮개 판: 현지 색을 아주 약하게 어둡게 하거나 채도를 낮춘다.
  4. 프리오퍼클 경계: 얇고 낮은 알파의 J자 soft line.
  5. 후방 opercular opening: 가장 선명하지만 폭은 좁은 dark arc.
  6. 목/몸통 분리: 별도 seam 또는 occlusion, 아가미덮개와 겹치지 않게 뒤쪽에 배치.

구현 메모

  • 조건은 if (is_head && operculum_enabled)로 제한한다.
  • operculum_ridge는 seam 폭보다 대비를 먼저 조절하게 한다.
  • 판 darken은 기본 0.92-0.88, 강한 프리셋도 0.82 아래로 잘 내려가지 않게 한다.
  • 후방 slit은 pu보다 너무 뒤로 가지 않게 한다. 몸통 seam과 합쳐지면 상처처럼 보인다.
  • 전방 seam은 au에서 단순 수직선이 아니라 위/아래가 살짝 뒤로 말리는 J자 곡률이 좋다.
  • 열림 값은 실제 판 전체 색을 키우는 값이 아니라 뒤쪽 arc 폭, 안쪽 어둠, 아주 얇은 rim에만 영향을 준다.
현재/스펙 값 보고서 권장 해석 튜닝 방향
operculum_up = vec2(0.0, 0.45) 측면 중앙에 넓게 걸친다. 기본은 0.34-0.40부터 보고, 큰 머리/납작한 어종에서만 키운다.
col * mix(0.80, 0.58, ridge) ridge가 높을수록 판 전체가 너무 어두워진다. mix(0.96, 0.84, ridge) 근처가 더 자연스럽다.
전방 seam 알파 0.18-0.45 작은 화면에서는 과하게 읽힐 수 있다. 0.10-0.28 정도로 낮추고, 후방 slit보다 약하게 둔다.
후방 slit 알파 0.5-0.85 방향은 맞다. 다만 폭이 커지면 상어 틈처럼 보일 수 있다. 알파는 유지하되 폭을 좁게, 열림 값에서만 폭 증가.
아가미덮개와 몸 분리 동시 처리 개념이 섞이면 덮개가 스티커처럼 튄다. 목 seam/ambient occlusion을 별도 블록으로 분리.

어종 간 형상차이를 구분하는 파라미터

연구 문헌에서 중요한 점은 “오퍼클 shape가 어종/생태와 함께 많이 변한다”는 것이다. 하지만 스프라이트 생성기에서는 실제 골격 전체를 재현하기보다, 사용자가 식별할 수 있는 5-6개 축으로 줄이는 편이 낫다.

축 시각 효과 어종/체형 힌트 추천 파라미터
길이 덮개가 볼 뒤에서 목 앞까지 얼마나 길게 닿는가. 일반적인 방추형 어류는 중간. 큰 머리/흡입섭식형은 상대적으로 넓게. operculum_length 또는 현재 operculum_size
높이 덮개가 등-배 방향으로 얼마나 큰가. 측편된 체형, 높은 체고의 어종은 높게. 길쭉한 어종은 낮고 길게. operculum_height
전방 경계 곡률 프리오퍼클이 곧은 선인지 J자/초승달인지. 농어류/시클리드류처럼 얼굴 구조가 또렷한 어종은 J자감이 강함. preopercle_curve
후방 출구 각도 slit이 수직인지 뒤로 기울었는지. 깊은 체형은 더 수직, 길쭉한 체형은 뒤로 누운 호가 자연스럽다. opercular_slit_slant
하단 막/branchiostegal 암시 아래쪽에 짧은 막 선이나 부드러운 그림자를 둔다. 큰 머리, 바닥성 어류, 근접 뷰에서 유용. 기본 스프라이트에서는 억제. branchiostegal_hint
가장자리 톱니/가시 프리오퍼클/오퍼클 가장자리에 미세한 톱니나 포인트. 일부 농어형/다람쥐고기류 등. 모든 어종에 일반화하면 지저분하다. opercle_serration, 기본 0

프리셋 방향

아키타입 형상 추천 값 감각
범용 방추형 중간 길이, 중간 높이, 후방 출구만 또렷함. length 1.0, height 0.9, ridge 0.25, slit alpha 중간.
시클리드/농어형 볼과 프리오퍼클 라인이 읽히며 덮개가 비교적 높다. length 1.05, height 1.1, curve 0.7, ridge 0.35.
잉어/금붕어형 둥근 머리와 넓은 덮개. 선은 부드럽고 색 차이는 약함. length 1.1, height 1.15, ridge 0.20, plate darken 낮음.
길쭉한 포식어형 낮고 긴 판. 후방 출구가 약간 뒤로 누움. length 1.2, height 0.75, slit slant 0.4, ridge 0.25.
소형 테트라/댄니오형 작고 거의 색 변화만 있는 덮개. 선은 최소. length 0.85, height 0.75, ridge 0.10-0.18.
메기류 머리 폭과 피부/판 느낌이 다르므로 단순 일반화 주의. 별도 head archetype과 결합. serration/spine은 기본값에서 끈다.

실행 가능한 다음 단계

  1. 즉시 완화: fish_body_toon.gdshader의 조건을 is_head && operculum_enabled로 바꾸고, plate darken과 전방 seam 알파를 낮춘다.
  2. 목 경계 분리: 아가미덮개 내부가 아니라 head/body 접합부에 얇은 neck occlusion을 별도 마스크로 추가한다.
  3. J자 경계: 전방 seam을 단순 세로선에서 위/아래가 뒤로 말리는 곡선 함수로 바꾼다.
  4. 프리셋 축 정리: 현재 operculum_size, height, open, ridge는 유지하되, 나중에 preopercle_curve와 slit_slant만 추가하면 어종 차이가 훨씬 잘 난다.
  5. 렌더 검증: side, 3/4, front, small sprite scale에서 확인한다. 정면에서는 덮개가 거의 사라지고, 측면/3/4에서만 자연스럽게 읽히는 것이 정상이다.
